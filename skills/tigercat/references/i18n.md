# i18n

Tigercat uses `locale` objects for component copy and `direction` metadata for RTL. Built-in locale IDs: `en-US`, `zh-CN`, `zh-TW`, `ja-JP`, `ko-KR`, `th-TH`, `vi-VN`, `id-ID`, `es-ES`, `fr-FR`, `de-DE`, `pt-BR`, `ar-SA`.

## Where Text Lives

| Area                 | Components / files                                                                                                       |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| Core types           | `packages/core/src/types/locale.ts`                                                                                      |
| Locale packs         | `packages/core/src/utils/i18n/locales/*`, `packages/core/src/utils/i18n/datepicker-locales/*`                            |
| Providers            | Vue and React `ConfigProvider`                                                                                           |
| Localized components | Pagination, Table pagination, DataTableWithToolbar, DatePicker, TimePicker, Modal, Drawer, Upload, FormWizard, TaskBoard |
| Derived components   | Kanban uses TaskBoard, so TaskBoard locale and drag labels apply unless overridden through TaskBoard-compatible props.   |

## Imports

Prefer subpath imports so bundlers only load the requested language.

```ts
import { zhCN } from '@expcat/tigercat-core/locales/zh-CN'
import { ZH_CN_DATEPICKER_LOCALE } from '@expcat/tigercat-core/datepicker-locales/zh-CN'
```

Vue and React use the same locale object shape.

```vue
<ConfigProvider :locale="zhCN">
  <App />
</ConfigProvider>
```

```tsx
<ConfigProvider locale={zhCN}>
  <App />
</ConfigProvider>
```

## Async Locale

`ConfigProvider` accepts a locale object, a Promise, or a loader function. During async load, `useTigerConfig()` exposes `localeLoading`; failures keep the previous or default locale.

```tsx
<ConfigProvider locale={() => import('@expcat/tigercat-core/locales/zh-CN')}>
  <App />
</ConfigProvider>
```

## Text Resolution

Per label, precedence is: dedicated text prop (e.g. Modal `okText`) → component `labels` prop → component `locale` prop → `ConfigProvider` locale → built-in default. When `locale.locale` starts with `zh`, FormWizard, Pagination/Table pagination, and TaskBoard use Chinese defaults for missing labels; otherwise they use English defaults.

Table and DataTableWithToolbar pagination inherit `ConfigProvider` locale. Set `pagination.locale` to override only pagination text, or set `pagination.locale: false` to disable pagination i18n and use explicit pagination text props or built-in English fallback text.

## No i18n (Custom Text)

Single-language apps can skip the locale system entirely and just supply their own wording — no locale IDs, no language-pack imports (smaller bundle). Two layers, both optional and backward compatible:

Per component — pass a flat `labels` object (the component's own text fields). Available on Modal, Drawer, Pagination, FormWizard, TaskBoard, Upload (DatePicker/TimePicker already have `labels`).

```tsx
<Pagination total={100} labels={{ totalText: '{total} results', prevPageAriaLabel: 'Previous' }} />
<Modal open showDefaultFooter labels={{ okText: 'Confirm', cancelText: 'Dismiss' }} />
```

Global — wrap the app once with `ConfigProvider`. Use `defineText(...)` to author a full text object from a flat overlay (alias of `defineLocale` without needing a `locale` code/`direction`):

```tsx
import { defineText } from '@expcat/tigercat-core'

const appText = defineText({ modal: { okText: 'Confirm' }, pagination: { totalText: '{total} items' } })

<ConfigProvider locale={appText}><App /></ConfigProvider>
```

A component `labels` prop overrides global text for that instance. `TigerText` is the flat type (`TigerLocale` minus `locale`/`direction`).

## RTL

`ar-SA` sets `direction: 'rtl'`. Providers sync this to `<html dir>` and `data-tiger-dir`; Pagination and DatePicker mirror navigation icons. Custom RTL languages should set `direction: 'rtl'` and use theme utility classes for custom layout that needs mirroring.

## Add A Locale

When adding a language, update these together:

| Need             | Files                                                                   |
| ---------------- | ----------------------------------------------------------------------- |
| Shared copy      | `packages/core/src/utils/i18n/locales/<id>.ts`                          |
| Date picker copy | `packages/core/src/utils/i18n/datepicker-locales/<id>.ts`               |
| Package exports  | `packages/core/package.json`                                            |
| Build entries    | `packages/core/tsup.config.ts`                                          |
| Tests            | `tests/core/i18n-locales.spec.ts`, `tests/core/datepicker-i18n.spec.ts` |

Validate with the focused locale tests plus `pnpm build` when export maps or build entries change.
