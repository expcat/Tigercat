**版本号**: 0.6.0
**发布计划**: 2025年8月

## 0. 版本概览

v0.6.0 是 Tigercat **第一个新增组件的版本**，标志着从架构优化转向功能扩展。本版本的核心目标是针对 B 端业务场景，补齐**表单录入能力**和**数据展示能力**。

**主要成果指标**:

- 新增 12-15 个组件
- 增强 15-20 个现有组件（v0.5.0 未涵盖的）
- Table 重大升级（虚拟滚动、高级筛选、行编辑等）
- Form 重大升级（动态字段、异步校验、联动规则等）
- 国际化扩展：所有新增组件支持 i18n

**预期版本后的组件总数**: 72-75 个（相比 v0.4.3 的 60+，增长 20%）

### A.2 新增组件的完整规格示例

以 **Cascader (级联选择)** 为例，展示新增组件应提供的完整规格：

#### Cascader - 级联选择器

**功能说明**:

Cascader 用于处理多级分类数据，用户逐级选择后返回完整的选择路径。常见于地区选择、部门选择、商品分类选择等场景。

**API 规格**:

```typescript
// @expcat/tigercat-core/types/props.ts

export interface CascaderOption {
  label: string
  value: string | number
  children?: CascaderOption[]
  disabled?: boolean
}

export interface CascaderProps {
  // 必需
  options: CascaderOption[]

  // 可选
  modelValue?: (string | number)[] // Vue
  value?: (string | number)[] // React
  placeholder?: string
  disabled?: boolean
  readonly?: boolean
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl'

  // 级联特定
  changeOnSelect?: boolean // 任何一级变化都触发 change 事件
  expandTrigger?: 'hover' | 'click' // 如何展开下级菜单
  multiple?: boolean // 支持多选
  maxTagCount?: number // 多选时最多显示几个标签
  checkStrictly?: boolean // 多选时，父子节点是否独立

  // 搜索
  showSearch?: boolean
  searchPlaceholder?: string
  filterOption?: (input: string, option: CascaderOption) => boolean

  // 外观
  clearable?: boolean
  suffixIcon?: string

  // 虚拟滚动（数据量大时性能优化）
  virtual?: boolean
  virtualHeight?: number
}

export interface CascaderEmits {
  'update:modelValue': [(value: (string | number)[]) => void]
  change: [(value: (string | number)[]) => void]
  blur: []
  focus: []
}

export interface CascaderSlots {
  default?: () => any // 自定义渲染级联路径
  option?: (data: { option: CascaderOption; index: number }) => any
}
```

## B. 重大升级：Table 组件

### B.1 升级目标

当前的 Table 已支持基础功能（排序、筛选、分页、固定表头等），但距离企业级标准还有差距。v0.6.0 需要在以下方面进行重大升级：

| 升级项       | 当前状态 | 目标状态                      | 业务价值               |
| ------------ | -------- | ----------------------------- | ---------------------- |
| 虚拟滚动     | 无       | 支持 10k+ 行数据不卡顿        | 大数据表格性能关键     |
| 行编辑       | 无       | 支持单行/批量编辑             | 数据录入主要场景       |
| 高级筛选     | 基础筛选 | 支持复杂条件、范围筛选        | B 端数据分析常见需求   |
| 列拖拽排序   | 无       | 支持鼠标拖拽调整列顺序        | 提升用户体验           |
| 自定义汇总行 | 无       | 支持合计、平均值等汇总        | 财务报表、数据分析常用 |
| 行分组       | 无       | 支持按某列值分组              | 复杂数据展示           |
| 导出功能     | 无       | 支持导出为 CSV、Excel（基础） | 数据分享、下载         |

### B.2 Table 新增 Props 规格

以下为 Table 的新增 props（在 v0.5.0 基础上追加）：

```typescript
interface TableProps<T = any> {
  // v0.5.0 基础
  columns: ColumnDefinition<T>[];
  data: T[];
  // ... 其他基础 props

  // v0.6.0 新增：虚拟滚动
  virtual?: boolean;
  virtualHeight?: number;
  virtualItemHeight?: number | ((index: number) => number);

  // v0.6.0 新增：行编辑
  editable?: boolean;
  editableCells?: Map<string, Set<number>>;  // 哪些单元格可编辑
  onCellChange?: (rowIndex: number, columnKey: string, newValue: any) => void;

  // v0.6.0 新增：高级筛选
  filterMode?: 'basic' | 'advanced';
  advancedFilterRules?: FilterRule[];
  onFilterChange?: (rules: FilterRule[]) => void;

  // v0.6.0 新增：列拖拽排序
  columnDraggable?: boolean;
  onColumnOrderChange?: (orderedColumns: ColumnDefinition<T>[]) => void;

  // v0.6.0 新增：汇总行
  summaryRow?: {
    show: boolean;
    data: Record<string, any | ((column: ColumnDefinition<T>) => any)>;
  };

  // v0.6.0 新增：行分组
  groupBy?: string;  // 按哪一列分组
  groupRender?: (groupKey: string, items: T[]) => VNode | ReactNode;

  // v0.6.0 新增：导出
  exportable?: boolean;
  exportFilename?: string;
}

interface FilterRule {
  column: string;
  operator: 'equals' | 'contains' | 'gt' | 'lt' | 'between' | ...;
  value: any;
  logic?: 'and' | 'or';
}
```

### C.3 使用示例

**Vue 动态字段示例**:

```vue
<template>
  <TiForm ref="formRef" v-model="formData" :rules="rules" :dynamic-fields="true">
    <!-- 静态字段 -->
    <TiFormItem label="姓名" name="name">
      <TiInput v-model="formData.name" />
    </TiFormItem>

    <!-- 动态字段容器 -->
    <div v-for="(field, idx) in dynamicFields" :key="field.id">
      <TiFormItem :label="field.label" :name="`phone[${idx}]`">
        <TiInput v-model="dynamicFields[idx].value" />
      </TiFormItem>
      <TiButton type="danger" @click="removeDynamicField(idx)"> 删除 </TiButton>
    </div>

    <TiButton @click="addDynamicField">添加电话</TiButton>
    <TiButton type="primary" @click="handleSubmit">提交</TiButton>
  </TiForm>
</template>

<script setup lang="ts">
import { ref } from 'vue'

const formRef = ref()
const formData = ref({ name: '', phone: [] })
const dynamicFields = ref<Array<{ id: string; label: string; value: string }>>([])

const addDynamicField = () => {
  const newField = {
    id: `phone-${Date.now()}`,
    label: `电话 ${dynamicFields.value.length + 1}`,
    value: ''
  }
  dynamicFields.value.push(newField)

  // 更新 form 规则
  formRef.value.addField({
    name: `phone[${dynamicFields.value.length - 1}]`,
    label: newField.label,
    rules: [{ required: true, message: '请输入电话号码' }]
  })
}

const removeDynamicField = (idx: number) => {
  dynamicFields.value.splice(idx, 1)
  formRef.value.removeField(`phone[${idx}]`)
}

const handleSubmit = async () => {
  const valid = await formRef.value.validate()
  if (valid) {
    console.log('Form data:', formData.value)
  }
}
</script>
```
