# v0.7.0 版本规格书：视觉升级与样式系统

**版本号**: 0.7.0
**发布计划**: 2025年10月
**预估工期**: 10-12 人月
**主题**: 全面提升视觉品质，从"能用"升级到"好看"，建立完整的主题系统

---

## 0. 版本概览

v0.7.0 是 Tigercat 的**视觉设计年**。在 v0.5.0 奠定设计系统基础、v0.6.0 完善功能的基础上，v0.7.0 将全面升级所有组件的视觉表现，推出 5+ 套预设主题，并优化暗色模式和响应式设计。

**主要成果指标**:

- 新增 8-12 个视觉/反馈类组件
- 现有 20-25 个组件进行视觉全面升级
- 推出 5 套完整的预设主题包
- 暗色模式全覆盖，所有组件都有优化的暗色视觉
- 响应式设计增强，支持 Container Queries
- 图表组件大幅增强（新增图表类型、交互优化）

**预期版本后的组件总数**: 85-95 个（相比 v0.6.0 的 72-75，增长 15%）

### A.3 主题应用方式

**使用预设主题**:

```html
<!-- HTML 方式：添加 class -->
<div class="theme-default">
  <button class="ti-button">Default 主题</button>
</div>

<div class="theme-vibrant">
  <button class="ti-button">Vibrant 主题</button>
</div>
```

**JavaScript 方式**:

```typescript
import { ThemeManager } from '@expcat/tigercat-core';

// 切换主题
ThemeManager.setTheme('vibrant');

// 获取当前主题
const currentTheme = ThemeManager.getCurrentTheme();

// 自定义主题
ThemeManager.defineTheme('custom', {
  colors: { primary: { 500: '#ff0000' }, ... },
  // ...
});
```

**Tailwind Config 方式**:

```javascript
// tailwind.config.js
import { tigercatPlugin } from '@expcat/tigercat-core/tailwind'

export default {
  plugins: [
    tigercatPlugin({
      theme: 'vibrant', // 默认使用 vibrant 主题
      themes: ['default', 'vibrant', 'professional', 'minimal', 'natural']
    })
  ]
}
```

## B. 组件视觉全面升级

### B.1 升级的主要维度

每个组件在以下维度进行升级（不是所有维度都适用于所有组件）：

| 维度             | 描述                                            | 影响组件                                    |
| ---------------- | ----------------------------------------------- | ------------------------------------------- |
| **颜色与对比度** | 优化颜色搭配，确保 WCAG AA 对比度标准           | 所有组件                                    |
| **间距与排版**   | 优化 padding、margin、font size、line height    | 所有组件                                    |
| **圆角统一**     | 使用 Token 中定义的标准圆角值（xs, sm, md, lg） | 所有组件                                    |
| **阴影与深度**   | 使用分层阴影营造视觉深度感                      | Card, Modal, Drawer, Dropdown, Select, etc. |
| **微交互**       | 添加 hover、focus、active 等状态的平滑过渡      | Button, Link, Input, Card, etc.             |
| **空状态**       | 优化无数据、加载中等特殊状态的视觉              | Table, List, Select, etc.                   |
| **禁用状态**     | 统一的禁用状态视觉（灰化、低对比度）            | 所有可交互组件                              |
| **加载动画**     | 使用统一的加载 spinner 设计                     | Modal, Form, Button, etc.                   |

## D. 暗色模式全覆盖

### D.1 当前状态与升级目标

**当前状态** (v0.4.3):

- 支持通过 CSS class `.dark` 切换暗色模式
- 某些组件的暗色适配不完整（颜色、对比度）
- 无统一的暗色模式色彩系统

**升级目标** (v0.7.0):

- 所有 60+ 现有组件 + 新增 8+ 组件都有完整的暗色视觉
- 实现自动暗色模式检测 (`prefers-color-scheme`)
- 建立完整的暗色模式色彩 Token

### D.2 暗色模式工作清单

| 2    | 全组件暗色视觉审查 | 逐个审查 60+ 现有组件，调整暗色下的样式    | 4d       |
| 3    | 自动暗色模式检测   | 实现 `prefers-color-scheme` 检测和自动切换 | 1d       |
