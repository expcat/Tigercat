## B. API 一致性全面审查

对所有 100+ 组件进行审查：

| 维度         | 检查项                                                 |
| ------------ | ------------------------------------------------------ |
| Props 命名   | 是否统一 (disabled, open, size, variant 等)            |
| Props 类型   | 类型定义是否完整、准确                                 |
| Events 命名  | Vue: kebab-case (@change), React: camelCase (onChange) |
| Slots 命名   | 是否统一 (default, header, footer, icon 等)            |
| 默认值合理性 | 所有 Props 默认值是否合理、安全                        |
| 文档一致性   | Props 表格、类型定义、示例代码是否一致                 |

### 自动化检查工具

建立 API 一致性扫描工具：

```typescript
validateComponentAPI(componentProps: Record<string, any>): ValidationResult {
  // 检查 1: props 命名规范
  // 检查 2: props 类型完整
  // 检查 3: 必需 props 文档注释
  return { invalid: [...], warnings: [...] }
}
```
