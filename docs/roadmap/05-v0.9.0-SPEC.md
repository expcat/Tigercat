# v0.9.0 版本规格书：生态完善与质量冲刺

**版本号**: 0.9.0
**发布计划**: 2026年2月
**预估工期**: 8-10 人月
**主题**: 为 v1.0.0 做最后的质量冲刺，完善生态工具链

---

## 0. 版本概览

v0.9.0 是冲刺 v1.0.0 发布的**最后一个 beta 版本**。这个版本的核心不是新增功能，而是通过多个维度的质量检查和改进，确保 Tigercat 达到企业级产品的标准。

**主要成果指标**:

- 补齐剩余的缺失组件（5-8 个）
- API 一致性全面审查，修复不一致的地方
- 无障碍 (a11y) 达到 WCAG 2.1 AA 全覆盖
- 文档站升级，完整的组件文档、API 自动生成
- 测试覆盖率冲刺到 95%+
- 完整的 v0.x → v1.0 迁移指南

**预期版本后的组件总数**: 105-115 个（相比 v0.8.0 的 95-105，增长 10%）

## B. API 一致性全面审查

### B.1 审查维度

对所有 100+ 组件进行以下维度的 API 审查：

| 维度         | 检查项                                                         | 预估工时 |
| ------------ | -------------------------------------------------------------- | -------- |
| Props 命名   | 所有 Props 是否遵循统一规范 (disabled, open, size, variant 等) | 3d       |
| Props 类型   | 所有 Props 的类型定义是否完整、准确                            | 2d       |
| Events 命名  | 所有 Events 是否遵循统一规范 (@change, @update, @click 等)     | 2d       |
| Slots 命名   | 所有 Slots 是否遵循统一规范 (default, header, footer, icon 等) | 1d       |
| 默认值合理性 | 所有 Props 的默认值是否合理、安全                              | 2d       |
| 文档一致性   | Props 表格、类型定义、示例代码是否一致                         | 3d       |

### B.2 审查工具

建立自动化工具扫描 API 不一致：

```typescript

  // ...
```

### F.2 Codemod 脚本示例

```typescript
// scripts/migrate-to-v1.0.js

const jscodeshift = require('jscodeshift')

// 示例 1: Props 命名迁移 (visible → open)
const migrateVisibleToProp = (fileInfo, api) => {
  const j = api.jscodeshift
  const root = j(fileInfo.source)

  // Vue: v-model="visible" → v-model="open"
  root
    .find(j.JSXAttribute, { name: { name: 'v-model' } })
    .filter((path) => path.value.value.value === 'visible')
    .forEach((path) => {
      path.value.value.value = 'open'
    })

  return root.toSource()
}

// 示例 2: Props 值迁移 (type → variant)
const migrateTypeToVariant = (fileInfo, api) => {
  const j = api.jscodeshift
  const root = j(fileInfo.source)

  // <Button type="primary" /> → <Button variant="primary" />
  root.find(j.JSXOpeningElement, { name: { name: 'Button' } }).forEach((path) => {
    path.value.attributes = path.value.attributes.map((attr) => {
      if (attr.name.name === 'type') {
        attr.name.name = 'variant'
      }
      return attr
    })
  })

  return root.toSource()
}

module.exports = {
  migrateVisibleToProp,
  migrateTypeToVariant
}
```
