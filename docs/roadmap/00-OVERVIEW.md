## 2. 版本间依赖关系

```
v0.4.3 (当前) → v0.5.0 (架构筑基)
                    ├─→ v0.6.0 (表单数据) ─┐
                    └─→ v0.7.0 (视觉升级) ─┤
                                            ├─→ v0.8.0 (高级交互)
                                            │       ↓
                                            └─→ v0.9.0 (质量冲刺) → v1.0.0 (发布)
```

**关键依赖**:

1. **v0.5.0 是基石** — Core 重构、Design Token、类型系统为所有后续版本提供基础
2. **v0.6.0 ↔ v0.7.0 低耦合** — 表单/数据能力和视觉升级互相增强但不阻塞
3. **v0.8.0 需要前序完成** — 依赖 v0.6.0 数据能力 + v0.7.0 视觉一致性
4. **v0.9.0 汇总冲刺** — 解决前序版本遗留的 API 一致性、a11y、文档问题
5. **v1.0.0 不新增功能** — 仅做最终验收和发布

## 6. v0.5.0 预计 Breaking Changes

1. **Props 命名统一**: `visible` / `isOpen` → `open`（影响 Modal, Drawer, Dropdown, Menu, Popover, Tooltip 等）
2. **类型导出调整**: 分散 types → 统一到 `@expcat/tigercat-core/types`
3. **CSS 变量命名**: 从 `--tiger-*` 扩展为语义层级 `--tiger-color-*` 等
4. **Core 包模块结构**: `chart-utils.ts` 拆分为独立模块

迁移指南将在 v0.5.0 发布时同步提供。
