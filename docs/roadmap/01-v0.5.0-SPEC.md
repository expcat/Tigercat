v0.5.0 是一次**架构级升级**，不新增组件，但增强 15–20 个现有组件。核心工作：

1. **Core 包重构** — 模块化、Headless 抽象、可独立测试
2. **Design Token 体系** — 从零散 CSS 变量升级到三层 Token 系统
3. **类型系统强化** — 泛型支持、严格模式、完整导出
4. **a11y 第一轮** — ARIA 属性补齐、键盘导航、焦点管理
5. **Props 命名统一** — 全库 API 一致性标准化
6. **动效系统抽象** — 进入/退出/过渡动效标准化

预期 **3–5 个 Breaking Changes**，需发布迁移指南。

### A.4 关键实现模式

#### 模式 1: Headless Floating Logic

```typescript
// core/utils/floating.ts — 框架无关

export interface FloatingOptions {
  placement: 'top' | 'bottom' | 'left' | 'right'
  offset: number
  trigger: 'hover' | 'click' | 'focus'
  delay?: number
  boundary?: Element
  autoUpdate?: boolean
}

export interface FloatingState {
  x: number
  y: number
  placement: string
  isOpen: boolean
  referenceEl: Element | null
  floatingEl: Element | null
}

export function useHeadlessFloating(options: FloatingOptions): {
  state: FloatingState
  update: () => void
}
```

**Vue 包装**:

```typescript
// vue/composables/useFloatingPopup.ts
export function useFloatingPopup(options: FloatingOptions) {
  const { state, update } = useHeadlessFloating(options)
  return { x: computed(() => state.x), y: computed(() => state.y) /* ... */ }
}
```

**React 包装**:

```typescript
// react/hooks/usePopup.ts
export function usePopup(options: FloatingOptions) {
  const [state, setState] = useState<FloatingState>(...)
  const { state: headlessState, update } = useHeadlessFloating(options)
  useEffect(() => { setState(headlessState) }, [headlessState])
  return { x: state.x, y: state.y /* ... */ }
}
```

#### 模式 2: 泛型组件类型

```typescript
// core/types/generics.ts

export interface TableProps<T = any> {
  columns: ColumnDefinition<T>[]
  data: T[]
}

export interface ColumnDefinition<T = any> {
  key: keyof T
  title: string
  width?: number
  sortable?: boolean
  render?: (value: T[keyof T], row: T) => ReactNode | VNode
}
```

#### 模式 3: 统一类型导出

```typescript
// core/types/index.ts
export type { ButtonProps, InputProps, SelectProps } from './props'
export type { ButtonEvents, InputEvents } from './events'
export type { TableProps, SelectProps, FormFieldProps } from './generics'

```

## B. Design Token 体系建设

### B.1 现状与目标

- 当前仅有 `--tiger-primary`, `--tiger-primary-hover` 等少量 CSS 变量
- 目标：完整三层 Token 体系 (Global → Alias → Component)
- 输出格式：JSON、CSS 变量、TypeScript、Tailwind 配置

详细 Token 规格见 [附录 B](./appendix/BCD-DESIGN-TOKEN-A11Y-TEMPLATE.md)。

### B.2 任务清单

| #   | Token 类别        | 任务                                                   | 验收标准                                        |
| --- | ----------------- | ------------------------------------------------------ | ----------------------------------------------- |
| 1   | 全局 Token - 颜色 | 主色、中性色、功能色 50-950 级色阶；JSON + CSS 变量    | 每种颜色完整色阶；CSS 变量应用到组件            |
| 2   | 全局 Token - 间距 | xs(2px) ~ 4xl(64px) 间距                               | 导出为 `--tiger-space-*`                        |
| 3   | 全局 Token - 圆角 | none, sm, md, lg, full 圆角值                          | 导出为 `--tiger-radius-*`                       |
| 4   | 全局 Token - 阴影 | xs ~ xl 阴影                                           | 导出为 `--tiger-shadow-*`                       |
| 5   | 全局 Token - 字体 | font-family, font-size, font-weight, line-height       | 完整字体 Token 体系                             |
| 6   | 全局 Token - 动效 | duration(快/中/慢), easing(in/out/in-out)              | 导出为 `--tiger-duration-*`, `--tiger-easing-*` |
| 7   | Alias Token       | 语义化映射（如 `--tiger-color-bg-primary` → blue-600） | 100+ 个 Alias Token                             |
| 8   | Component Token   | 组件级 Token（button height, padding, radius 等）      | 所有 60+ 组件都有 Component Token               |
| 9   | Token JSON 导出   | 生成 `tokens.json`, `tokens.ts`, `tokens.css`          | 三种格式一致导出                                |
| 10  | Tailwind 插件集成 | 更新 tigercatPlugin 应用 Token                         | `tailwind.config.js` 中 Token 自动应用          |

## D. 动效系统

### D.1 现状与目标

- 各组件动效标准不一致（CSS transition / Vue Transition / React CSSTransition 混用）
- 无统一进入/退出/过渡规范
- `prefers-reduced-motion` 支持不完整

### D.2 动效规范

动效 Token 已定义于 Token 体系：

| 动效名           | Duration | Easing   | 适用场景                            |
| ---------------- | -------- | -------- | ----------------------------------- |
| fade-enter       | 200ms    | ease-out | 元素淡入（通用）                    |
| fade-leave       | 200ms    | ease-in  | 元素淡出                            |
| slide-up-enter   | 300ms    | ease-out | 从下往上滑入（Modal, Notification） |
| slide-down-leave | 300ms    | ease-in  | 从上往下滑出                        |
| scale-enter      | 200ms    | ease-out | 缩放显示（Popover, Tooltip）        |
| collapse-enter   | 300ms    | ease-out | 展开（Accordion, Collapse）         |
| collapse-leave   | 300ms    | ease-in  | 收起                                |

### D.3 实现指引

```typescript
// core/composables/useTransition.ts
export type TransitionType = 'fade' | 'slide-up' | 'slide-down' | 'scale' | 'collapse'

export interface TransitionConfig {
  type: TransitionType
  duration?: 'fast' | 'base' | 'slow'
  reduceMotion?: boolean // 尊重 prefers-reduced-motion
}

export function getTransitionClasses(config: TransitionConfig): {
  enterActiveClass: string
  leaveActiveClass: string
}

function prefersReducedMotion(): boolean {
  return window.matchMedia('(prefers-reduced-motion: reduce)').matches
}
```

### D.4 组件迁移清单

| 组件            | 迁移方案                                       |
| --------------- | ---------------------------------------------- |
| Modal           | `useTransition(type: 'fade')`                  |
| Drawer          | `useTransition(type: 'slide-up')` + 自定义方向 |
| Collapse        | 新增 `useTransition(type: 'collapse')`         |
| Dropdown        | 新增 scale 动效                                |
| Tooltip/Popover | 新增 scale 动效                                |
