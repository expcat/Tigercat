# v0.5.0 版本规格书：架构筑基与核心增强

**版本号**: 0.5.0
**发布计划**: 2025年6月
**预估工期**: 10-12 人月
**主题**: 强化架构基础，完善设计系统，为后续快速迭代铺路

---

## 0. 版本概览

v0.5.0 是 Tigercat 的一次**重要的、非增量式的架构升级版本**。与其他版本不同，v0.5.0 的核心不是添加新组件（新增0个，但增强15-20个现有组件），而是：

1. **重构 Core 包** — 将分散的底层逻辑集中、模块化、可复用
2. **建立完整的 Design Token 体系** — 从零散的 CSS 变量升级到三层 Token 系统
3. **强化类型系统** — 支持泛型、严格模式、完整导出
4. **第一轮 a11y 审查与改进** — 为 WCAG 2.1 AA 达成做准备
5. **统一 Props 命名与语义** — 全库范围内 API 一致性标准化
6. **动效系统抽象** — 将进入/退出/过渡动效标准化

预期本版本将带来**3-5 个较大的 breaking changes**，需要发布详细的迁移指南。

## A. Core 包重构

### A.1 目标与现状

**现状分析**:

- `@expcat/tigercat-core/utils/chart.ts` 单文件 2400+ 行，包含所有图表工具函数
- 浮动弹出层逻辑分散于 `useFloatingPopup` (Vue) 和 `usePopup` (React) 中，有重复代码
- 类型定义分散，无统一导出入口
- Headless 逻辑与具体 UI 实现混杂

**目标**:

- 将核心包分为清晰的、可复用的、可独立测试的模块
- 建立 Headless 抽象层，供 Vue 和 React 两端使用
- 完整的、树可摇的(tree-shakeable)模块导出结构
- 测试覆盖率达到 90%+

**模式 1: Headless Floating Logic**

当前浮动弹出层的 Vue 实现和 React 实现各有一份 hook，代码重复。v0.5.0 需要：

```typescript
// core/utils/floating.ts - Headless 实现，框架无关

export interface FloatingOptions {
  placement: 'top' | 'bottom' | 'left' | 'right' | ...;
  offset: number;
  trigger: 'hover' | 'click' | 'focus';
  delay?: number;
  boundary?: Element;
  autoUpdate?: boolean;
}

export interface FloatingState {
  x: number;
  y: number;
  placement: string;
  isOpen: boolean;
  referenceEl: Element | null;
  floatingEl: Element | null;
}

/**
 * Headless floating logic
 * 返回计算后的 x, y 坐标，由具体的 UI 层（Vue/React）渲染
 */
export function useHeadlessFloating(
  options: FloatingOptions
): {
  state: Reactive<FloatingState>;        // Vue style
  setState: (s: FloatingState) => void;  // React style
  update: () => void;
} {
  // 使用 @floating-ui/dom 的原始 API，计算坐标
  // 返回 state 和 update 函数，让 Vue/React 按各自的方式响应
}

// vue/composables/useFloatingPopup.ts - Vue 包装

export function useFloatingPopup(options: FloatingOptions) {
  const { state, update } = useHeadlessFloating(options);
  // Vue reactive 包装
  return {
    x: computed(() => state.x),
    y: computed(() => state.y),
    // ...
  };
}

// react/hooks/usePopup.ts - React 包装

export function usePopup(options: FloatingOptions) {
  const [state, setState] = useState<FloatingState>(...);
  const { state: headlessState, update } = useHeadlessFloating(options);

  useEffect(() => {
    setState(headlessState);
  }, [headlessState]);

  return { x: state.x, y: state.y, ... };
}
```

## B. Design Token 体系建设

### B.1 目标与现状

**现状分析**:

- 当前仅有基础的 CSS 变量 `--tiger-primary`, `--tiger-primary-hover`, 等少量变量
- 无完整的三层 Token 体系 (Global → Alias → Component)
- 颜色系统不完整（仅有主色、次色、功能色，无完整色阶）
- 间距、圆角、阴影、字体、动效无统一定义

**目标**:

- 建立完整的三层 Token 体系
- 颜色系统：50-950 完整色阶（主色、中性色、功能色）
- 间距、圆角、阴影、字体系统完整定义
- 输出完整的 JSON、CSS 变量、Tailwind 配置文件

## C. 现有组件批量增强（第一批）

### C.1 增强组件选择原则

选择使用频率最高、最基础、改进效果最直观的 15-20 个组件进行深度增强：

**高优先级增强名单**:

1. Button — size/variant 完善、loading 状态优化、icon 位置支持
2. Input — prefix/suffix/clearable 增强、character count、show password
3. Select — 搜索优化、多选增强、自定义选项渲染、virtual scroll 基础
4. Form — 异步校验、动态字段、表单级别的 loading、disabled
5. Table — 排序优化、筛选优化、固定表头性能优化、简化 API
6. Modal/Drawer — 拖拽移动（v0.8.0）、自定义尺寸、z-index 管理
7. Card — 布局变体增加、padding 可定制
8. Avatar — size/shape 完善、fallback 优化
9. Badge — 位置定位优化
10. DatePicker/TimePicker — 交互改进、快捷选择、范围选择增强
11. Tree — 搜索过滤、虚拟滚动基础、拖拽排序基础
12. Tabs — 更多变体、滚动支持、禁用状态
13. Pagination — 各种 UI 变体
14. Alert/Notification — 关闭动画、多操作按钮支持
15. Grid/Space — 可响应式配置增强

### C.2 每个组件的增强项（示例：Button）

**组件**: Button

**当前 API**:

```typescript

  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl'

```

**增强需求**:

| #   | 增强项               | 当前          | 增强后                                                     | 验收标准                                      |
| --- | -------------------- | ------------- | ---------------------------------------------------------- | --------------------------------------------- |
| 1   | Loading 状态         | 无            | 新增 `loading?: boolean`; loading 时显示 spinner，禁用交互 | loading=true 时，按钮被禁用，右侧显示 spinner |
| 2   | Icon 位置            | 固定右侧      | 支持 `iconPosition?: 'left' \| 'right'`                    | 可通过 prop 控制 icon 位置                    |
| 3   | Block 模式           | 无            | 新增 `block?: boolean`，100% 宽度                          | block=true 时宽度为 100%                      |
| 4   | Type 属性            | 无            | 新增 `htmlType?: 'button' \| 'submit' \| 'reset'`          | 生成正确的 HTML button type                   |
| 5   | Icon 作为独立 slot   | 固定单个 slot | 拆分为 `#icon` slot，分离 icon 和文字                      | 可通过 slot 自定义 icon 位置和内容            |
| 6   | Danger 变体          | 无            | 新增 `danger?: boolean`，自动应用红色 variant              | danger=true 时应用 error 颜色                 |
| 7   | Group 包装           | 无            | 提供 `<ButtonGroup>` 组件用于排列多个按钮                  | 按钮之间间距一致、可选分组样式                |
| 8   | Ghost + loading 组合 | 无            | loading 时自动调整样式避免闪烁                             | Ghost 按钮在 loading 时保持视觉一致           |

## D. 动效系统

### D.1 目标与现状

**现状**:

- 各组件动效标准不一致，有的用 CSS transition，有的用 Vue `<Transition>`/React `<CSSTransition>`
- 无统一的进入、退出、过渡动效规范
- `prefers-reduced-motion` 支持不完整

**目标**:

- 统一的动效规范：进入(enter)、退出(leave)、过渡(transition)
- 所有组件遵循同一套动效 Token (duration, easing)
- 完整支持 `prefers-reduced-motion` (无障碍)
- 提供 Vue 和 React 的动效工具函数库

### D.2 动效规范

**动效 Token** (已定义于 Token 体系中):

```json
{
  "duration": {
    "fast": "100ms", // 快速交互（ripple、tooltip 显示）
    "base": "200ms", // 标准（modal 显示、切换、淡入淡出）
    "slow": "300ms", // 缓慢（展开/收起、drawer 滑动）
    "slower": "500ms" // 非常缓慢（无）
  },
  "easing": {
    "in": "cubic-bezier(0.4, 0, 1, 1)", // 加速
    "out": "cubic-bezier(0, 0, 0.2, 1)", // 减速
    "in-out": "cubic-bezier(0.4, 0, 0.2, 1)" // 加速-减速
  }
}
```

**动效类型定义**:

| 动效名           | Duration | Easing   | 适用场景                               |
| ---------------- | -------- | -------- | -------------------------------------- |
| fade-enter       | 200ms    | ease-out | 元素淡入显示                           |
| fade-leave       | 200ms    | ease-in  | 元素淡出隐藏                           |
| slide-up-enter   | 300ms    | ease-out | 从下往上滑入（Modal, Notification 等） |
| slide-down-leave | 300ms    | ease-in  | 从上往下滑出                           |
| scale-enter      | 200ms    | ease-out | 缩放显示（Popover, Tooltip 等）        |
| collapse-enter   | 300ms    | ease-out | 展开（Accordion, Collapse 等）         |
| collapse-leave   | 300ms    | ease-in  | 收起                                   |

### D.3 Copilot 实现指引

**Vue 动效工具** (`core/composables/useTransition.ts`):

```typescript
import { CSSProperties } from 'vue'

export type TransitionType = 'fade' | 'slide-up' | 'slide-down' | 'scale' | 'collapse'

export interface TransitionConfig {
  type: TransitionType
  duration?: 'fast' | 'base' | 'slow'
  reduceMotion?: boolean // 尊重 prefers-reduced-motion
}

export function getTransitionClasses(config: TransitionConfig): {
  enterActiveClass: string
  leaveActiveClass: string
} {
  const { type, duration = 'base', reduceMotion } = config

  if (reduceMotion || prefersReducedMotion()) {
    return { enterActiveClass: '', leaveActiveClass: '' }
  }

  // 返回 Tailwind 动效 class names
  // 例如: 'transition-all duration-200 ease-out'
  // (需要在 Tailwind config 中定义这些 class)
}

function prefersReducedMotion(): boolean {
  return window.matchMedia('(prefers-reduced-motion: reduce)').matches
}
```

**React 动效工具** (`core/hooks/useTransition.ts`):

```typescript
export interface TransitionStyles {
  enterFrom: CSSProperties
  enterTo: CSSProperties
  leaveFrom: CSSProperties
  leaveTo: CSSProperties
  transition: CSSProperties
}

export function getTransitionStyles(config: TransitionConfig): TransitionStyles {
  if (config.reduceMotion || prefersReducedMotion()) {
    return {
      /* all instant */
    }
  }

  const duration = DURATION_MAP[config.duration]
  const easing = EASING_MAP[config.type]

  return {
    enterFrom: getEnterFrom(config.type),
    enterTo: getEnterTo(config.type),
    leaveFrom: getLeaveFrom(config.type),
    leaveTo: getLeaveTo(config.type),
    transition: { transition: `all ${duration} ${easing}` }
  }
}
```

### D.4 组件迁移

所有使用 Transition/CSSTransition 的组件需要迁移到新的动效系统：

| 组件            | 当前动效            | 迁移方案                                          | 预估工时 |
| --------------- | ------------------- | ------------------------------------------------- | -------- |
| Modal           | Transition          | 使用 useTransition(type: 'fade')                  | 0.5d     |
| Drawer          | Transition (自定义) | 使用 useTransition(type: 'slide-up') 并自定义方向 | 1d       |
| Collapse        | 无动效              | 新增 useTransition(type: 'collapse')              | 1d       |
| Dropdown        | 无动效              | 新增使用 scale 动效                               | 0.5d     |
| Tooltip/Popover | 无动效              | 新增使用 scale 动效                               | 0.5d     |

## E. 无障碍 (a11y) 第一轮审查与改进

### E.1 目标与现状

- 键盘导航支持不完整
- 焦点管理策略不统一

**目标 (v0.5.0)**:

- 审查所有 60+ 现有组件，识别 a11y 缺陷

- 不要求完全达到 WCAG 2.1 AA（这是 v0.9.0 的目标），但为达成做准备

## H. 破坏性变更与迁移方案

### H.1 主要 Breaking Changes

**1. Props 命名统一** (影响度: 🔴 高)

```
visible → open                    (所有显示/隐藏相关的 prop)
isOpen → open
isDisabled → disabled
isLoading → loading
type → variant                    (在有歧义时，统一用 variant 表示样式变体)
```

**影响组件**: Modal, Drawer, Dropdown, Menu, Popover, Tooltip, Collapse, etc. (20+ 个)

**迁移方案**:

- v0.5.0 发布时，保持 deprecated warning
- 在 v0.5.x 期间，同时支持旧 props（内部转换到新 props）
- v0.6.0 移除旧 props

**2. 类型导出调整** (影响度: 🟡 中)

```
// Before
import { ButtonProps } from '@expcat/tigercat-vue/components/Button';

// After
import type { ButtonProps } from '@expcat/tigercat-core/types';
```

**影响**: TypeScript 用户的导入语句

**3. CSS 变量扩展** (影响度: 🟡 中)

```css
/* Before */
--tiger-primary: #0ea5e9;

/* After */
--tiger-color-primary-500: #0ea5e9;
--tiger-space-xs: 2px;
```

旧变量仍存在（向后兼容），但建议迁移到新变量。

### H.2 迁移指南的组成

单独的 `MIGRATION_v0.5.0.md` 文件应包含：

1. **Breaking Changes 总结** — 列表和影响面积
2. **逐项迁移指南** — 对每个 breaking change，给出具体的迁移步骤
3. **Codemod 脚本** — 自动化脚本帮助用户升级（至少处理 Props 命名）
4. **FAQ** — 常见问题和解答
5. **社区支持** — GitHub Issue 标签、讨论区链接
