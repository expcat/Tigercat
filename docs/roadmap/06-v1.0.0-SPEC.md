v1.0.0 是 Tigercat 的第一个正式版本，标志着从实验阶段进入稳定阶段。

---

## A. 稳定性保证

### A.1 SemVer 承诺

| 版本范围       | 变更类型                     | 破坏性 |
| -------------- | ---------------------------- | ------ |
| v1.x.x (patch) | Bug 修复、内部优化           | 否     |
| v1.y.0 (minor) | 新特性、新组件、向后兼容增强 | 否     |
| v2.0.0 (major) | 破坏性变更、API 重设计       | 是     |

### A.2 API 稳定性承诺（v1.x 系列）

1. 组件 Props 不会移除或改变行为
2. Events 名字、参数类型不变
3. Slots 名字不变
4. 所有新增 Props 都有合理默认值
5. `@expcat/tigercat-core` 公开 API 不变
6. CSS 变量名（主题 Token）保持一致

Breaking changes 处理：必须发布 v2.0.0，提前 3 个月通知，提供迁移指南和自动化工具。

## C. 发布流程

### C.1 发布前 (T-7d)

- [ ] 冻结代码，仅接受 bug 修复
- [ ] CI/CD 全绿
- [ ] CHANGELOG 完整（breaking changes + 新特性 + bug fixes）
- [ ] 更新所有文档版本号

### C.2 发布审查 (T-3d)

- [ ] 代码审查：所有 breaking changes、新 API 经 senior review
- [ ] 文档审查：全部文档经编辑审查
- [ ] 国际化审查：所有语言翻译经审查
- [ ] 安全审查：依赖无已知漏洞

### C.3 发布日

```bash
git status                    # 确认 main 分支无本地改动
npm version 1.0.0
npm run build
npm publish
```

- 创建 GitHub Release (tag: v1.0.0)
- 部署文档站到生产
- 验证所有链接有效、代码示例可运行
